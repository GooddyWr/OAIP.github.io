ЛР 9 Что такое подпрограмма? Зачем она нужна?
Что такое подпрограмма?
Подпрограмма — это именованный блок кода, который можно вызвать из разных мест программы. В Pascal бывают двух видов: procedure (процедура) и function (функция).
Зачем использовать подпрограммы?
Три главных причины:
1. Устранение повторений — один раз написал, вызываешь сколько угодно.
2. Структурирование — задача делится на подзадачи, программу легче читать.
3. Тестирование — ошибку легче найти и исправить, если код разбит на части.
Чем процедура отличается от функции?
Функция возвращает значение — его можно использовать в выражении. Процедура значение не возвращает — просто выполняет действие. Используй функцию, когда нужен результат; процедуру — когда нужно что-то сделать или вернуть несколько значений через параметры var.
ЛР 9 Синтаксис объявления процедуры и функции
Как выглядит объявление процедуры?
procedure DoAdd(a, b: real; var res: real);
begin
    res := a + b;   { res передан по ссылке — оригинал изменится }
end;
Как выглядит объявление функции?
function Add(a, b: real): real;   { после : указан тип возвращаемого значения }
begin
    Add := a + b;    { вариант 1: присваиваем имени функции }
    { Result := a + b;  вариант 2: используем Result }
end;
Где должна быть объявлена подпрограмма?
До места первого вызова — в разделе описаний программы. Компилятор читает сверху вниз и должен "знать" о подпрограмме до её использования.
ЛР 9 Параметры: передача по значению и по ссылке
Что такое формальные параметры?
Переменные, указанные в заголовке подпрограммы. При вызове им передаются реальные значения (фактические параметры). Формальные — это "заглушки", фактические — настоящие данные.
Что значит передача по значению?
Внутрь подпрограммы передаётся копия переменной. Любые изменения не затрагивают оригинал. Это поведение по умолчанию — без ключевого слова.
Что значит передача по ссылке (var)?
Перед параметром стоит var. Передаётся адрес переменной. Изменения внутри подпрограммы изменяют оригинал. Используется, когда нужно вернуть несколько значений или изменить входной параметр. 
procedure Swap(var a, b: integer);
var t: integer;
begin
    t := a; a := b; b := t;
end;
{ после вызова Swap(x, y) — x и y реально поменяются местами }
Что такое параметры const?
Параметр передаётся по ссылке (эффективно), но изменить его внутри нельзя — компилятор запретит. Используется для оптимизации при передаче больших структур.
ЛР 9 Result, возврат значения, рекурсия
Как вернуть значение из функции?
Два способа: присвоить значение имени функции или специальной переменной Result. Оба работают одинаково.
Чем Result лучше имени функции?
Result можно использовать в выражениях (например, if Result > 0 then), а имя функции в выражении трактуется как рекурсивный вызов. 
function MinElement(a: array of real): real;
var i: Integer;
begin
    Result := a[0];
    for i := 1 to Length(a)-1 do
        if a[i] < Result then
            Result := a[i];  { используем в выражении — ок }
end;
Прерывается ли функция при присвоении Result?
В Pascal — НЕТ! Функция продолжает выполняться даже после Result := .... Это важное отличие от C, Java, Python, где return немедленно прерывает выполнение.
Что такое рекурсия?
Вызов функции из самой себя. Обязательно должно быть условие выхода (база рекурсии), иначе бесконечный цикл. 
function Factorial(n: integer): integer;
begin
    if n <= 1 then Result := 1
    else Result := n * Factorial(n - 1);
end;
Что будет, если не присвоить Result?
Функция вернёт непредсказуемое значение (мусор из памяти). Это частая ошибка.
ЛР 10 Виды файлов в Pascal
Чем текстовый файл отличается от бинарного?
Текстовый — данные хранятся как строки символов. Число 42 будет записано как два символа '4' и '2'. Удобно читать в блокноте. Бинарный — данные хранятся так же, как в памяти компьютера. Число 42 занимает ровно 2/4 байта в двоичном виде. Быстрее, компактнее, но нечитаем "глазами".
Какие виды файловых переменных есть в Pascal?
ОбъявлениеТипПрименение
var f: TextТекстовыйСтроки, числа как текст
var f: file of integerТипизированный бинарныйОднородные данные (все Integer, все Real...)
var f: fileНетипизированный бинарныйПроизвольные байты, копирование файлов
Нельзя писать file of string, так как string имеет переменную длину. Нужно создать тип: type Str20 = string[20]; и использовать его.
ЛР 10 Процедуры открытия, закрытия, работы с файлом
Что такое Assign и зачем он нужен?
Связывает файловую переменную с именем файла на диске. Без этого шага с файлом нельзя работать. Вызывается первым, до любого открытия. 
Assign(f, 'output.txt');  { связать f с файлом output.txt }
Чем Reset отличается от Rewrite?
ПроцедураДействиеОсобенность
Reset(f)Открыть для чтенияДля текстовых — только чтение. Для бинарных — чтение И запись. Файл должен существовать!
Rewrite(f)Открыть для записиСоздаёт новый файл. Если файл уже есть — УНИЧТОЖАЕТ его содержимое!
Append(f)Открыть для дозаписиТолько для текстовых. Добавляет в конец, не удаляет старое.
Что такое Close и почему важно его вызывать?
Закрывает файл. Файловая система блокирует файл пока он открыт — другие программы не могут его использовать. Также без Close данные из буфера могут не записаться на диск. Всегда закрывай файл после окончания работы с ним.
Что делает Flush?
Принудительно записывает содержимое буфера на диск, не закрывая файл. Используется если нужно гарантировать запись данных до закрытия.
ЛР 10 Чтение, запись, навигация по файлу
Как прочитать файл строчка за строчкой?
var f: text; s: string;
begin
    Assign(f, 'file.txt');
    Reset(f);
    while not Eof(f) do
    begin
        Readln(f, s);     { читаем строку }
        writeln(s);
    end;
    Close(f);
end;
Что такое Eof и Eoln?
Eof(f) — возвращает true, если достигнут конец файла. Используется в цикле при чтении. Eoln(f) — возвращает true, если достигнут конец строки в текстовом файле.
Что такое SeekEof и SeekEoln?
Аналоги Eof и Eoln, но сначала пропускают пробельные символы (пробелы, табуляции). Удобны при чтении чисел, разделённых пробелами.
Что такое Seek, FilePos, FileSize?
Для бинарных файлов: Seek(f, n) — перемещает указатель на элемент с номером n (прямой доступ). FilePos(f) — возвращает текущую позицию указателя. FileSize(f) — количество элементов в типизированном файле (или байт в нетипизированном).
Что делает Truncate?
Обрезает файл начиная с текущей позиции указателя — всё после неё удаляется. Работает только с бинарными файлами.
Как переименовать или удалить файл?
Rename(f, 'newname.txt') — переименовывает файл, связанный с переменной. Erase(f) — удаляет файл. Важно: файл при этом должен быть закрыт.
ЛР 10 Типизированный файл — примеры и типичные ошибки
Как работать с типизированным файлом?
var f: file of integer; num: integer; i: integer;
begin
    Assign(f, 'nums.dat');
    Rewrite(f);                      { создаём файл }
    for i := 1 to 10 do
        Write(f, i * i);             { пишем квадраты }
    Close(f);

    Reset(f);                        { открываем для чтения }
    while not Eof(f) do
    begin
        Read(f, num);
        Write(num, ' ');
    end;
    Close(f);
end.
Что произойдёт если обратиться к файловой переменной после Close?
Ошибка выполнения! После закрытия память, связанная с файлом, освобождена. То же самое если обратиться до открытия — память ещё не выделена.
ЛР 11 Записи (record)
Что такое запись (record)?
Пользовательский тип данных — набор полей разных типов под одним именем. Как паспорт: в одной структуре хранится имя (string), возраст (integer), пол (перечислимый тип).
Как объявить и использовать запись?
type
    Person = record
        Name: string;
        Age, Weight: integer;
    end;

var p: Person;
begin
    p.Name := 'Иванов';    { доступ через точку }
    p.Age  := 20;
end.
Что такое оператор with и зачем он нужен?
Позволяет обращаться к полям записи без указания имени переменной — сокращает код при работе с несколькими полями подряд. 
with p do
begin
    Name := 'Васильева';
    Age  := 22;
end;
{ Эквивалентно: p.Name := 'Васильева'; p.Age := 22; }
Можно ли присвоить одну запись другой?
Да. Оператор d := p скопирует все поля из p в d, если они одного типа.

Зачем нужна динамическая память?
Когда размер данных неизвестен заранее. Например, список книг: добавляешь по одной — память выделяется по мере необходимости. Массив с заранее заданным размером был бы неудобен.
ЛР 11 Указатели: объявление, Nil, разыменование
Что такое указатель?
Переменная, которая хранит не значение, а адрес ячейки памяти, где лежит значение. Как почтовый адрес — сам адрес не является содержимым квартиры, но позволяет до неё добраться.
Как объявить указатель?
var
    p: ^Integer;      { типизированный — указывает только на Integer }
    q: Pointer;       { нетипизированный — может указывать на что угодно }
Что такое Nil?
Специальное значение указателя — "не указывает никуда". Аналог null. Используется для обозначения конца списка, начального состояния или пустого результата поиска. Проверка: if p = nil then ...
Как получить значение по указателю (разыменование)?
Через символ ^ после имени указателя: p^ — это значение, на которое указывает p. 
var p: ^integer;
begin
    New(p);     { выделить память }
    p^ := 42;   { записать значение }
    writeln(p^); { прочитать значение → 42 }
    Dispose(p); { освободить память }
end.
Какие три состояния может иметь указатель?
1. Не инициализирован — содержит мусор, обращение опасно. 2. Содержит адрес — после New(p) или p := @x. 3. Nil — явно обнулён, ни на что не указывает.
ЛР 11 New, Dispose, GetMem, FreeMem, Mark, Release
Что делает New(p)?
Выделяет блок памяти под переменную того типа, на который указывает p, и сохраняет адрес в p. После этого можно работать с p^.
Что делает Dispose(p)?
Освобождает память, занятую p^. После вызова значение p становится неопределённым. Если не вызвать — будет утечка памяти (память занята, но недоступна).
Чем GetMem отличается от New?
GetMem(p, Size) выделяет ровно Size байт и возвращает нетипизированный указатель. Используется когда нужен точный контроль над размером. New выделяет столько, сколько нужно для типа автоматически.
Что такое Mark и Release?
Mark(p) — запоминает текущее состояние динамической памяти (сохраняет "точку"). Release(p) — освобождает всю память, выделенную после вызова Mark. Удобно для массового освобождения памяти. 
Mark(p);        { запомнить точку }
New(p1); New(p2);
Release(p);     { освободить p1 и p2 сразу }
Что такое MemAvail и MaxAvail?
MemAvail — возвращает общий объём свободной динамической памяти в байтах. MaxAvail — размер самого длинного непрерывного свободного участка.
Что такое Addr и операция @?
Оба возвращают адрес переменной: p := Addr(x) и p := @x — одно и то же. Позволяют указателю "смотреть" на уже существующую статическую переменную.
ЛР 11 Однонаправленный список
Что такое однонаправленный список?
Структура данных, где каждый элемент (узел) содержит данные и указатель на следующий элемент. Перемещение только в одну сторону — от первого к последнему. Последний элемент указывает на nil.
Как устроен узел списка?
type
    PBook = ^TBook;       { указатель на TBook }
    TBook = record
        title: string[100];
        author: string[50];
        next: PBook;        { ссылка на следующий элемент }
    end;
Как добавить элемент в конец списка?
New(newNode);            { создать новый узел }
newNode^.data := value;
newNode^.next := nil;    { он последний }
{ найти последний элемент и связать: }
current := head;
while current^.next <> nil do
    current := current^.next;
current^.next := newNode;
Почему список лучше массива, если размер неизвестен?
Массив требует заранее объявить максимальный размер. Список позволяет добавлять элементы бесконечно — память выделяется под каждый узел отдельно по мере необходимости.
Что такое утечка памяти и как её избежать?
Утечка памяти — память выделена (New), но не освобождена (Dispose). Память занята, но программа к ней уже не обращается. При удалении узла из списка обязательно вызывай Dispose на удаляемый узел, а НЕ просто перезаписывай указатель.
ЛР 12 Что такое модуль (unit) и зачем он нужен
Что такое модуль?
Отдельно компилируемая программная единица, содержащая типы, константы, переменные, процедуры и функции. Модуль позволяет один раз написать код и подключать его в любые программы. Ключевое слово: unit.
Зачем нужны модули?
1. Повторное использование — написал один раз, используешь везде.
2. Сокрытие реализации — пользователь видит только интерфейс, детали скрыты в Implementation.
3. Структурирование — большая программа делится на логические части.
4. Ускорение компиляции — каждый модуль компилируется отдельно.
Как подключить модуль?
program MyProgram;
uses MathUtils, SysUtils;   { перечислить через запятую }
begin
    { теперь доступны все функции из этих модулей }
end.
ЛР 12 Структура модуля: Interface, Implementation, Initialization, Finalization
Из каких частей состоит модуль?
Unit ИмяМодуля;

Interface
    { Публичные объявления — видны снаружи }
    { Только заголовки процедур и функций, без тел }

Implementation
    { Реализация + скрытые (приватные) переменные и подпрограммы }

Initialization
    { Выполняется ОДИН РАЗ при загрузке модуля (необязательно) }

Finalization
    { Выполняется при завершении программы (необязательно) }

End.
Что такое раздел Interface?
Публичная часть — всё что видно снаружи. Здесь объявляются типы, константы, переменные и заголовки процедур/функций (без тела). Это "контракт" — что модуль предоставляет пользователю.
Что такое раздел Implementation?
Закрытая часть — реализация всех функций из Interface. Также может содержать дополнительные переменные и процедуры, которые не видны снаружи (приватные). Заголовки функций можно писать в сокращённой форме, если совпадают с Interface.
Что делает Initialization?
Выполняется один раз при загрузке модуля (когда программа стартует). Используется для начальной настройки: инициализация переменных, открытие файлов, выделение памяти. Необязателен.
Что делает Finalization?
Выполняется при завершении программы. Используется для "уборки": закрытие файлов, освобождение памяти, сохранение данных. Необязателен.
ЛР 12 Пример модуля + видимость, конфликты имён
Как выглядит простой модуль?
Unit MathUtils;

Interface
    function Factorial(n: integer): int64;
    function IsPrime(n: integer): boolean;

Implementation

    function Factorial(n: integer): int64;
    begin
        if n <= 1 then Result := 1
        else Result := n * Factorial(n-1);
    end;

    function IsPrime(n: integer): boolean;
    var i: integer;
    begin
        Result := n > 1;
        for i := 2 to Trunc(Sqrt(n)) do
            if n mod i = 0 then Result := false;
    end;

End.
Что если два модуля содержат функцию с одинаковым именем?
Используется квалифицированное имя: ИмяМодуля.Функция. Собственные идентификаторы модуля имеют приоритет над внешними. 
MathUtils.Factorial(5);
StatUtils.Mean(arr);
Нужно ли компилировать модуль перед использованием?
Да. Модуль должен быть предварительно скомпилирован. В Lazarus это происходит автоматически при сборке проекта, если файл модуля (.pas) находится в папке проекта.

ЛР 9 Операция MOD и DIV
Что такое MOD и для чего он нужен?
MOD — операция получения остатка от деления целых чисел. Например: 10 mod 3 = 1 (10 делится на 3 с остатком 1). Часто используется для проверки делимости (n mod 2 = 0 → чётное), работы с цифрами числа, цикличных структур.
Что такое DIV?
DIV — целочисленное деление (без остатка). Например: 10 div 3 = 3. В паре с MOD позволяет "разобрать" число на цифры: 542 div 10 = 54, 542 mod 10 = 2 (последняя цифра).
