ЛР 9 Рисование рамки — разбор кода по шагам
Зачем выносить рисование рамки в процедуру DrawFrame?
Для читаемости и переиспользования — можно вызвать несколько раз не копируя код. Процедура без параметров выполняет фиксированное действие.
Почему первый цикл рисует строку от 1 до 60, а средний цикл идёт от 2 до 19?
Первая строка (верхняя граница) уже нарисована первым циклом. Цикл средних строк начинается со второй строки. Заканчивается на 19 — последняя строка (нижняя граница) рисуется отдельным циклом.
Зачем внутри среднего цикла вложенный цикл for j := 2 to 59?
Позиция 1 занята левым краем (*), позиция 60 — правым. Между ними (2–59) выводятся 58 пробелов — внутреннее пространство рамки.
Почему здесь write, а не writeln при выводе внутреннего пространства?
Строка ещё не закончена — после пробелов нужно вывести правый край *. writeln преждевременно перешёл бы на новую строку.
Чем процедура DrawRect(width, height) лучше DrawFrame?
DrawRect принимает параметры — размер рамки задаётся при вызове. DrawFrame рисует рамку фиксированного размера 60×20. DrawRect универсальнее.
Что такое локальные переменные i, j внутри процедуры?
Локальные переменные существуют только во время работы процедуры и недоступны за её пределами. Глобальные переменные (например, w, h) существуют на протяжении всей программы.
Что такое фактические параметры в вызове DrawRect(w, h)?
Фактические параметры — реальные значения, передаваемые при вызове (здесь w и h). Формальные параметры (width, height) — имена в объявлении процедуры. w передаётся в width, h — в height.
ЛР 9 Функции для треугольника и трапеции
Что такое тип возвращаемого значения и как он указывается?
После двоеточия в заголовке функции: function TriPerimeter(a, b, c: real): real; — функция возвращает тип real. Значение возвращается присваиванием имени функции: TriPerimeter := a + b + c;
Почему нельзя использовать ^ для возведения в степень в Pascal?
В Pascal оператор ^ используется для работы с указателями. Для квадрата используется умножение (h*h) или функция Sqr(h).
Что такое формула Герона и как она используется в TriArea?
Формула для площади треугольника по трём сторонам: сначала вычисляется полупериметр s = (a+b+c)/2, затем площадь Sqrt(s*(s-a)*(s-b)*(s-c)). Переменная s — локальная внутри функции.
Почему в TrapPerimeter вызывается функция TrapSide, а не считается заново?
Чтобы не дублировать код. Функция уже написана и возвращает значение, которое можно использовать в выражении: TrapPerimeter := a + b + 2 * TrapSide(a, b, h);
Что такое :0:2 при выводе вещественного числа?
Форматирование вывода: :0:2 — минимальная ширина поля 0 (минимально необходимая), 2 знака после запятой. Применяется ко всему выражению, в том числе к сумме двух вызовов функции.
ЛР 9 Функция HasRealRoots и проверка палиндрома
Что такое дискриминант и как функция возвращает boolean?
Дискриминант: b*b - 4*a*c. Функция HasRealRoots присваивает результат сравнения: HasRealRoots := (b*b - 4*a*c) >= 0;. Если D ≥ 0 — вещественные корни есть, возвращается true.
Почему if HasRealRoots(a, b, c) then, а не if HasRealRoots(a,b,c) = true then?
Функция уже возвращает boolean — сравнение с true избыточно и просто удлиняет код. if HasRealRoots(...) then читается напрямую как «если есть вещественные корни».
Как работает проверка палиндрома через Str и сравнение символов?
Str(n, s) переводит число в строку. Затем цикл for i := 1 to Length(s) div 2 сравнивает симметричные символы: s[i] и s[Length(s)-i+1]. Если хотя бы одна пара не совпадает — isPalin := false.
Почему в функции IsPalindrome нужна промежуточная переменная isPalin?
Нельзя присваивать имени функции внутри цикла — в Pascal это может трактоваться как рекурсивный вызов. Используется отдельная переменная, которой в конце присваивается: IsPalindrome := isPalin;
Что такое алгоритм Евклида для нахождения НОД?
Цикл while b <> 0 do begin a := a mod b; a := a xor b; b := a xor b; a := a xor b; end;. Когда b становится 0, в a находится НОД. Три строки с xor — обмен значениями без дополнительной переменной.
Как найти НОД трёх чисел если функция принимает только два?
Вложенный вызов: GCD(GCD(a, b), c) — сначала находим НОД(a,b), потом НОД результата и c. Функция принимает два параметра, поэтому нужно два вызова.
ЛР 10 Текстовый файл — запись строк (Task1)
Что такое файловая переменная и чем f: text отличается от f: file of integer?
Файловая переменная — посредник между программой и файлом. text — текстовый файл, данные читаются/пишутся как строки (число 42 = два символа '4','2'). file of integer — бинарный файл, число хранится в двоичном виде.
Почему нельзя после assign сразу писать в файл?
assign только связывает переменную с именем файла. Нужно ещё открыть файл через rewrite (для записи) или reset (для чтения).
Что произойдёт если не вызвать close после записи?
Данные из буфера могут не записаться на диск. Файл останется заблокированным — другие программы не смогут его открыть.
Почему writeln(f, ...) записывает в файл, а не на экран?
Наличие файловой переменной f первым аргументом указывает куда идёт запись. Без f — вывод на экран. Аналогично для read(f, ...) — чтение из файла, не с клавиатуры.
Как правильно записать несколько строк и затем прочитать их обратно?
Сначала rewrite(f) → запись → close(f). Затем reset(f) → чтение → close(f). Повторный assign перед reset не нужен — связь уже установлена.
ЛР 10 Чтение чисел из файла — min, max, сумма (Task2)
Почему первое число читается до цикла отдельно?
Чтобы инициализировать minVal и maxVal реальным числом из файла. Если инициализировать нулём — результат будет неверным если все числа в файле положительные (минимум покажет 0, которого нет в файле).
Почему avg объявлена как real, а не integer?
При делении суммы на количество может получиться дробное число. В Pascal оператор / всегда возвращает вещественный результат, даже если оба операнда целые.
Что такое :0:2 после avg при выводе?
Форматированный вывод вещественного числа: 0 — минимальная ширина, 2 — два знака после запятой. Без форматирования вещественное число выводится в научной нотации (например 1.98E+001).
Чем read(f, num) отличается от readln(f, num) при чтении из файла?
read читает значение и оставляет указатель на той же строке. readln — читает и переходит на следующую строку. При чтении чисел из файла используется read.
ЛР 10 Типизированный файл и навигация — Seek, FileSize (Task4)
Как добавить число в конец типизированного файла не используя append?
append работает только для текстовых файлов. Для типизированных: открыть через reset(f), затем Seek(f, FileSize(f)) — перемещает указатель за последний элемент, затем write(f, num).
Что делает Randomize и что будет если его не вызвать?
Randomize инициализирует генератор случайных чисел на основе текущего времени. Без него каждый запуск программы будет давать одинаковую последовательность чисел.
Что вернёт Random(200) - 100?
Random(200) возвращает число от 0 до 199. После вычитания 100 диапазон становится от -100 до 99 — числа могут быть отрицательными. 200 в результат НЕ включается.
Зачем в функции FindNumber возвращается -1 если число не найдено?
-1 — сигнальное значение, которое не может быть реальным индексом (нумерация начинается с 0). При проверке результата: if foundPos >= 0 then — проверяем >= 0, потому что нулевой индекс тоже валиден.
Чем вызов функции отличается от вызова процедуры?
Функция возвращает значение — вызов стоит в правой части присваивания: foundPos := FindNumber(filename, searchNum). Процедура вызывается как команда: CreateFile(filename, n).
Почему pos mod 10 = 0 даёт перенос строки каждые 10 чисел?
mod возвращает остаток от деления. Когда pos кратно 10 (10, 20, 30...) — остаток равен 0 и вызывается writeln. Числа выводятся по 10 в строке для читаемости.
ЛР 11 Связный список — добавление и обход (HomeLibrary)
Почему PBook объявляется до TBook, хотя ссылается на него?
В Pascal в блоке type разрешено опережающее объявление — можно написать указатель на тип, который будет описан ниже в том же блоке. Это единственный случай, где порядок объявления не важен.
Зачем в записи TBook есть поле next: PBook?
Это указатель на следующую книгу в списке — именно это поле делает структуру связным списком. У последней книги next = nil — признак конца списка.
Почему head в процедуре AddBook передаётся через var?
Если список пуст — head нужно изменить (присвоить адрес первой книги). Без var изменения не передались бы обратно — head снаружи остался бы nil.
Чем current := head и current^.next := newBook решают разные ситуации?
head := newBook — список был пуст, первая книга становится головой. current^.next := newBook — список не пуст, находим последний элемент и прикрепляем новый в конец.
Почему цикл while current^.next <> nil проверяет next, а не current?
Нужно остановиться на последнем элементе (у которого next = nil), чтобы прикрепить к нему новую книгу. Проверка current <> nil вышла бы за последний элемент.
Почему libHead инициализируется nil в начале программы?
Без этого в libHead будет мусорный адрес. Проверка if head = nil покажет что список не пуст, хотя он пуст — поведение непредсказуемо.
ЛР 11 Удаление из списка — Dispose, prev (HomeLibraryWithDelete)
Зачем нужна переменная prev при удалении из списка?
Связный список односторонний — нельзя пойти назад. prev хранит предыдущий элемент, чтобы «перешить»: prev^.next := current^.next — предыдущий элемент теперь указывает на следующий после удаляемого.
Почему первый элемент удаляется отдельно от остальных?
При удалении первого элемента меняется сам head. У первого нет предыдущего, поэтому общий алгоритм с prev не подходит.
Зачем сохранять адрес в toDelete перед вызовом Dispose?
Сначала нужно перешить указатели, потом освободить память. Если освободить раньше — потеряем доступ к полю next удаляемого элемента. После Dispose нельзя обращаться к освобождённой памяти.
Что такое const перед параметром bookTitle?
Передача по ссылке без возможности изменения. Компилятор запрещает изменять параметр внутри процедуры. Для строк это выгоднее чем передача по значению — не создаётся копия.
ЛР 11 Поиск нечётного, очистка памяти — FindFirstOdd, FreeList
Почему FindFirstOdd возвращает PNode, а не Integer?
Возвращая указатель на узел, мы даём доступ ко всему узлу. Nil как результат удобно проверять: if result <> nil. Чтобы получить значение: result^.value.
Что делает функция Odd() в Pascal?
Встроенная функция: возвращает true если число нечётное, false если чётное. Проверяет последний бит числа. Эквивалентна value mod 2 <> 0, но короче и читабельнее.
Зачем Exit после FindFirstOdd := current?
Нашли первый нечётный — дальше искать не нужно. Без Exit цикл продолжит работу и перезапишет результат следующим нечётным элементом — вернёт последний, а не первый.
Почему в FreeList нельзя сначала вызвать Dispose, а потом head := head^.next?
После Dispose память освобождена — обращение к head^.next вызовет ошибку. Правильный порядок: сохранить адрес в temp → сдвинуть head → вызвать Dispose(temp).
Что такое string[100] и чем отличается от просто string?
string без размера — до 255 символов. string[100] ограничивает максимум 100 символами и занимает ровно 101 байт (100 + 1 байт для длины). Экономит память когда максимальная длина известна.
ЛР 12 Модуль MathUtils — factorial, IsPrime, PrintNumberInfo
Почему в разделе interface функции без тела, а тела в implementation?
Interface — публичные объявления (заголовки). Implementation — скрытый код. Программа-пользователь видит только interface — «контракт» что есть в модуле, но не детали реализации.
Почему результат Factorial имеет тип int64, а не integer?
Факториал быстро растёт: 20! уже больше 2 миллиардов, что не помещается в integer. int64 вмещает числа до 9.2×10¹⁸.
Почему res := 1 перед циклом в Factorial, а цикл начинается с 2?
Факториал 0 и 1 равен 1. Если n=0 или n=1, цикл не выполнится и вернётся 1 — правильный результат. Умножение на 1 ничего не меняет, поэтому начинаем с 2 для экономии.
Почему в IsPrime цикл идёт пока i * i <= n, а не i <= n?
Если у числа n есть делитель больше √n, то обязательно есть и делитель меньше √n. Проверять только до √n достаточно — это быстрее. Для n=100: проверяем до 10 вместо 100.
Зачем PrintNumberInfo вызывает IsPrime и Factorial внутри себя?
Процедура использует другие функции того же модуля — они доступны без дополнительных объявлений. Это избавляет от дублирования кода.
Зачем выносить функции в модуль, а не писать всё в program?
Модуль можно использовать в других программах без копирования кода. Это называется повторное использование кода. Также скрывает детали реализации и ускоряет компиляцию.
ЛР 12 Модуль StatUtils — динамический массив, Mean, Variance, StdDev
Что такое TRealArray = array of real?
Динамический массив — его размер задаётся во время работы программы через SetLength, а не при объявлении. Тип объявлен в interface — программа-пользователь может объявлять переменные этого типа.
Зачем SetLength(arr, n) и что будет без него?
SetLength задаёт размер динамического массива. Без него массив будет нулевой длины — запись в arr[i] вызовет ошибку выхода за границы.
Чем Length(a) отличается от High(a) для массива?
Length — количество элементов. High — последний индекс. Для массива из 5 элементов: Length=5, High=4. Динамические массивы индексируются с 0.
Почему uses Math стоит в implementation, а не в interface?
Модуль Math используется только в реализации (для Sqrt, Sqr). Снаружи он не нужен. Это скрывает детали реализации — пользователь модуля не обязан знать что внутри используется Math.
Почему StdDev вызывает Variance, а не считает сам?
СКО — это просто корень из дисперсии: StdDev := Sqrt(Variance(a)). Повторное использование уже написанной функции — не нужно дублировать формулу.
ЛР 12 Модуль StringUtils — ReverseStr, IsPalindrome, WordCount, ToAlternateCase
Почему в ReverseStr цикл идёт downto 1, а не to?
Нужно идти с конца строки к началу. downto уменьшает счётчик. Строки в Pascal индексируются с 1, последний символ имеет индекс Length(s).
Зачем в IsPalindrome удаляются пробелы перед проверкой?
Чтобы «А роза упала на лапу Азора» тоже считался палиндромом — пробелы не должны мешать. Также применяется LowerCase — 'А' и 'а' считаются одинаковыми.
Что такое переменная inWord: boolean в функции WordCount?
Флаг — отслеживает находимся ли мы внутри слова. Счётчик увеличивается только при переходе «не в слове → в слово» (первый символ слова). Это позволяет не считать одно слово несколько раз.
Что вернёт WordCount для строки ' hello world '?
2 — два слова. Пробелы по краям и несколько пробелов между словами не влияют на результат, так как считается переход из «не в слове» в «в слове».
Что делает ToAlternateCase и как определяется чётность символа?
Возвращает строку где нечётные символы (1, 3, 5...) в нижнем регистре, чётные (2, 4...) — в верхнем. Проверка чётности: i mod 2 = 0. Для 'Hello' вернёт 'hElLo'.
Можно ли написать if IsPalindrome(s) = true then?
Да, но это избыточно. Функция уже возвращает boolean, который сам является условием. Короче и читабельнее просто if IsPalindrome(s) then.
