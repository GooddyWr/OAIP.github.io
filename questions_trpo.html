ЛР7-8 ТРПО Визуальное программирование в Lazarus — основы
Что такое визуальное программирование?
Берёшь готовые компоненты из библиотеки LCL, перетаскиваешь на форму, настраиваешь свойства и пишешь обработчики событий. Каждый компонент — это класс: TLabel, TButton, TEdit и т.д.
Как создать новый проект в Lazarus?
Меню → Проект → Создать проект → Приложение. Каждый пример — отдельный проект в отдельной папке!
Что такое Инспектор объектов?
Панель в Lazarus. Две вкладки: Properties (свойства) и Events (события). Двойной клик по событию — Lazarus автоматически создаёт пустой обработчик.
Чем свойства отличаются от событий?
Свойства — внешний вид и состояние: размер, цвет, текст, доступность. События — поведение: что происходит при клике, вводе, открытии формы. Обработчик события — процедура, которую пишет программист.
Компоненты бывают видимыми и невидимыми?
Да. Форма при проектировании — контейнер для компонентов. На неё можно поместить и невидимые компоненты (например, таймеры, диалоги).

Пример: управление видимостью и доступностью BitBtn
Форма: Button1 «Показать кнопку», Button2 «Сделать доступной», BitBtn1 (Kind=bkClose, Enabled=False, Visible=False). 
{ Button1.OnClick — показать }
procedure TForm1.Button1Click(Sender: TObject);
begin
    BitBtn1.Visible := true;
end;

{ Button2.OnClick — разблокировать }
procedure TForm1.Button2Click(Sender: TObject);
begin
    BitBtn1.Enabled := true;
end;
После запуска BitBtn1 не видна. Button1 — кнопка появляется но серая. Button2 — становится активной и закрывает приложение.
ЛР7-8 ТРПО TSpeedButton — быстрая кнопка с фиксацией
Что такое TSpeedButton и чем отличается?
Страница Additional. Используется в панелях инструментов. Главная особенность — умеет фиксироваться в нажатом состоянии.
СвойствоЧто делает
GroupIndex0 = обычная кнопка; >0 = фиксируется при нажатии
AllowAllUpTrue = повторный клик отпускает кнопку
DownTrue = кнопка сейчас в нажатом состоянии
Как сделать группу взаимоисключающих кнопок?
Дать нескольким TSpeedButton одинаковый GroupIndex > 0 и AllowAllUp = true — работают как радиокнопки: нажатие одной отпускает предыдущую. 
{ Проверить состояние }
if SpeedButton1.Down then
    WriteLn('SpeedButton1 нажата');

{ Сбросить все кнопки из кода }
SpeedButton1.Down := false;
SpeedButton2.Down := false;

Что такое TLabeledEdit?
Страница Additional. Тот же TEdit, но со встроенной подписью — удобно когда над каждым полем нужен поясняющий текст. 
{ Задать подпись через Инспектор:
  раскрыть свойство EditLabel → Caption }

{ Или из кода: }
LabeledEdit1.EditLabel.Caption := 'Первое число';
Как обработать нажатие Enter в поле ввода?
Событие OnKeyPress. Параметр Key — код нажатой клавиши. Код Enter = #13. 
procedure TForm1.Edit1KeyPress(Sender: TObject; var Key: char);
begin
    if Key <> #13 then exit;  { не Enter — сразу выходим }
    { сюда попадаем только при нажатии Enter }
    MyProcedure;
end;
Проще установить у кнопки Default = True — тогда Enter = клик. Но при нескольких кнопках сработает та, у которой фокус. Надёжнее — OnKeyPress.
ЛР7-8 ТРПО Преобразование типов: строка ↔ число
Почему нужно преобразовывать типы?
Edit.Text — всегда строка. Для математики нужно преобразовать в число. Для вывода результата — обратно в строку.
Какие функции преобразования использовать?
ФункцияЧто делаетПример
StrToInt(s)Строка → IntegerStrToInt('42') = 42
StrToFloat(s)Строка → DoubleStrToFloat('3.14') = 3.14
IntToStr(n)Integer → строкаIntToStr(42) = '42'
FloatToStr(x)Double → строкаFloatToStr(3.14) = '3.14'
Если пользователь введёт буквы — StrToInt/StrToFloat выбросят исключение и программа упадёт. Всегда оборачивай в try..except!
var a, b: Integer;
begin
  try
    a := StrToInt(LabeledEdit1.Text);
    b := StrToInt(LabeledEdit2.Text);
    LabeledEdit3.Text := IntToStr(a + b);
  except
    ShowMessage('Введите корректное число!');
  end;
end;

ЛР7-8 ТРПО Обработка исключений: try..except
Что такое исключение?
Ошибка во время выполнения программы. Без обработки — программа аварийно завершается. Причины: неверный ввод пользователя, деление на ноль, несуществующий файл, переполнение, выход за пределы массива.
Синтаксис try..except
try
    { код, который может вызвать ошибку }
    a := StrToInt(Edit1.Text);
    b := a div StrToInt(Edit2.Text);  { деление на ноль? }
except
    { сюда попадаем при любой ошибке в try }
    ShowMessage('Ошибка! Проверьте введённые данные.');
end;
Если в try возникает ошибка — выполнение немедленно прыгает в except. Остаток кода в try не выполняется.
ЛР7-8 ТРПО Пример: авторизация с 3 попытками
Как работает программа?
Компоненты: TEdit, TButton, TLabel. Пароль задаётся в FormCreate. 3 попытки — на 3-й отказ и закрытие. Глобальные переменные объявляются в interface (иначе обработчики их не увидят). 
var
    Form1: TForm1;
    passw: string[4];
    Stroka: string;
    n: integer;  { счётчик попыток }
Зачем выносить логику в отдельную процедуру?
Чтобы не дублировать код в обработчике кнопки и обработчике Enter. Одна процедура avtorization — два места её вызывают. 
procedure TForm1.avtorization;
begin
    if passw <> Edit1.Text then
    begin
        Edit1.SetFocus;
        if n = 1 then begin
            ShowMessage('Неправильный пароль!'); inc(n);
        end else if n = 3 then begin
            ShowMessage('Вам отказано в доступе'); Close();
        end else begin
            Stroka := 'Вы ' + IntToStr(n) + ' раза ввели неправильный пароль';
            Stroka := Stroka + ' После 3-й попытки вам будет отказано в доступе';
            ShowMessage(Stroka); inc(n);
        end;
    end else begin
        ShowMessage('Вы успешно авторизовались!'); Close();
    end;
end;

{ Кнопка и Enter — оба вызывают одно и то же }
procedure TForm1.Button1Click(Sender: TObject);
begin  avtorization;  end;

procedure TForm1.Edit1KeyPress(Sender: TObject; var Key: char);
begin
    if Key <> #13 then exit;
    avtorization;
end;
ЛР7-8 ТРПО Пример: калькулятор с TSpeedButton и TStatusBar
Компоненты формы
3× TLabeledEdit (подписи: «Первое число», «Второе число», «Результат»), 4× TSpeedButton (+, −, ×, ÷), 1× TBitBtn (закрыть), 1× TStatusBar. Для всех SpeedButton: GroupIndex=1, AllowAllUp=true.
Код операций (один паттерн для всех)
{ SpeedButton1 «+» }
procedure TForm1.SpeedButton1Click(Sender: TObject);
begin
    LabeledEdit3.Text := IntToStr(
        StrToInt(LabeledEdit1.Text) + StrToInt(LabeledEdit2.Text));
    StatusBar1.SimpleText := 'Сложение';
    LabeledEdit1.SetFocus;
    LabeledEdit1.SelectAll;
end;

{ SpeedButton4 «÷» — целочисленное деление div! }
procedure TForm1.SpeedButton4Click(Sender: TObject);
begin
    LabeledEdit3.Text := IntToStr(
        StrToInt(LabeledEdit1.Text) div StrToInt(LabeledEdit2.Text));
    StatusBar1.SimpleText := 'Деление нацело';
    LabeledEdit1.SetFocus;
    LabeledEdit1.SelectAll;
end;
div — целочисленное деление. 7 div 2 = 3. Для вещественного используй / и StrToFloat.
Как переходить между полями по Enter?
Событие OnKeyPress на LabeledEdit1. При #13 — переводим фокус и сбрасываем SpeedButton: 
procedure TForm1.LabeledEdit1KeyPress(Sender: TObject; var Key: char);
begin
    if Key = #13 then begin
        LabeledEdit2.SetFocus;
        SpeedButton1.Down := false;
        SpeedButton2.Down := false;
        SpeedButton3.Down := false;
        SpeedButton4.Down := false;
        exit;
    end;
end;
ЛР7-8 ТРПО ООП в коде: TSquare и TCube
Иерархия и методы классов

Как TCube использует inherited?
FSide объявлено private в TSquare — в TCube оно недоступно напрямую. Поэтому сторона восстанавливается через inherited Perimeter / 4. 
{ Площадь поверхности: 6 граней }
function TCube.Area: Double; override;
begin
    Result := 6 * inherited Area;  { вызывает TSquare.Area = a² }
end;

{ Объём: a³ }
function TCube.Volume: Double;
begin
    Result := Power(inherited Perimeter / 4, 3);
    { Perimeter = 4a → /4 = a → a³ }
end;
